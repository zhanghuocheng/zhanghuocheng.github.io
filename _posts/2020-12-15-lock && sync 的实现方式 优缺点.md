### lock && sync 的实现方式 优缺点

 

|      | 实现方式                              | 是否可尝试 | 是否可重入 | 公平/非公平 | 是否可中断 | 异常是否                              | 资源竞争不激烈 | 资源竞争激烈 |
| ---- | ------------------------------------- | ---------- | ---------- | ----------- | ---------- | ------------------------------------- | -------------- | ------------ |
| lock | 通过AQS实现 基于接口                  | 是         | 是         | 是          | 是         | 因此使用Lock时需要在finally块中释放锁 |                | 常态         |
| sync | 基于对象内存空间  enter/exit实现 内置 | 否         | 是         | 否          | 否         | 自动释放                              | 稍差           | 特差         |
